# 保姆级教程：接入飞书使用

把这个 Agent 接进飞书，让你在飞书里就能维护经历库、生成简历和逐字稿。

> ⚠️ 说明：飞书（Lark）的 AI 能力随版本/企业权限不同，产品名可能是「飞书智能伙伴」「Aily 应用」「Base 智能助手」等。本文给出通用思路 + 两种落地方案，具体按钮以你的飞书版本为准。核心不变：**SKILL.md 当指令，经历库当知识源**。

---

## 它在飞书里怎么映射

| 本 Agent | 飞书里的对应物 |
|---|---|
| `SKILL.md`（行为说明） | 智能伙伴的「人设与指令 / 提示词」 |
| `templates/`、`经历库/`、`materials/` | 飞书**知识库（Wiki）**文档，或**多维表格（Base）**记录 |
| 生成结果 `output/` | 新建飞书云文档 |
| AI 执行 | 飞书智能伙伴 / Base 智能助手 |

---

## 方案 A（推荐）：知识库 + 智能伙伴

**适合**：想最快用起来、像在 Obsidian 里一样对话生成。

### 步骤一：把结构搬进飞书知识库
1. 飞书 → **云文档** → **知识库** → 新建一个知识库，命名「求职文档 Agent」
2. 在知识库里按文件夹结构建分组页面：`模板`、`经历库`、`原始素材`、`产出`
3. 把本仓库的内容粘进去：
   - `templates/逐字稿/产品面试话术模版.示例.md` → 知识库「模板」下新建文档粘贴
   - 你的简历模板 → 「模板」下
   - 经历 → 先粘进「原始素材」

### 步骤二：创建智能伙伴并写入指令
1. 飞书 → 找到 **智能伙伴 / AI 伙伴**（部分版本在「效率」或工作台里）→ **创建**
2. 把本仓库 `SKILL.md` 的全文，粘进智能伙伴的 **指令/提示词** 框
   - 注意把 SKILL.md 里「文件路径」相关描述，改成「在本知识库中查找对应页面」
3. **绑定知识库**：在智能伙伴的「知识 / 数据源」里，关联第一步建的知识库

### 步骤三：使用
直接和智能伙伴对话：
- 「把这段项目经历整理进经历库」→ 它在知识库「经历库」分组新建/更新文档
- 「写一份突出搜索经验的简历」→ 它读经历库 + 模板，产出文档
- 「写小红书项目的逐字稿，简单版+复杂版」

> 飞书智能伙伴能否「写回」知识库取决于版本权限。若不能自动写回，它会把整理好的内容输出在对话里，你手动粘进知识库页面即可。

## 方案 B：lark-cli skill（高级 / 开发者）

如果你用 lark-cli 工具链，可把本 SKILL.md 封装成一个 lark skill：
- 经历库用**飞书多维表格**存储，通过 `lark-base` 读写
- 模板和产出用**飞书云文档**，通过 `lark-doc` 读写
- 用 lark 的 skill 机制把 SKILL.md 注册为可调用 skill

这条路最贴合「飞书原生」，但需要 lark-cli 使用基础，普通用户用方案 A 即可。

---

## 跨平台一致性的关键

无论 Obsidian 还是飞书，**SKILL.md 是同一份**——它是平台无关的纯文本指令。你只需：
1. 在该平台准备好「存档案的地方」（Obsidian 文件夹 / 飞书知识库或 Base）
2. 把 SKILL.md 交给该平台的 AI（Claudian / 飞书智能伙伴）当指令
3. 按相同的对话方式使用

经历库的内容也可以在两个平台间复制同步，做到一处维护、处处可用。
