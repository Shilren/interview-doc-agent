# 使用指南

## 日常工作流

```
① 加素材   把项目经历（任何格式）粘进 materials/
② 整理     对 AI 说「整理进经历库」→ 沉淀成 经历库/ 里的完善档案
③ 生成     说「写简历」/「写某项目逐字稿」/「按这份 JD 定制」
④ 取用     产出在 output/，按需微调
```

## 三类产出怎么要

### 写简历
- 「按 templates/简历 的风格生成一份简历」
- 「生成一份突出**搜索策略**经验的简历」（侧重某方向）
- 「把简历控制在一页」

### 写面试逐字稿
- 「写**小红书项目**的逐字稿，要简单版和复杂版」
- 「写一段 2 分钟的自我介绍逐字稿」
- 「把这几个项目串成一份完整自我介绍（3 点匹配度结构）」

### 按 JD 定制
1. 把目标岗位 JD 粘进 `jd/`（新建一个文件，如 `jd/字节-增长PM.md`）
2. 「参照 jd/ 里的字节增长 PM，生成匹配的简历」
3. AI 会挑出与 JD 最相关的经历、突出对应数据、弱化无关项

## 经历整理的质量标准

AI 整理 materials → 经历库时，会保证每篇档案包含：
- **一句话**：这段经历最有力的概括
- **三段式**：背景判断 / 关键问题 / 手段&成果
- **手段 why+how**：逻辑是什么 + 具体动作（触发时机/面向谁/展示什么）
- **量化数据速取**：所有数字集中列出，面试高频引用
- **判断力与人设亮点**：体现「为什么这样判断」+ 可迁移优势

## 面试逐字稿的方法论（内置）

- 三段式：**背景思考 → 定义问题 → 手段&成果**
- 固定句式：「我判断发力重点在 X」「逻辑是…具体动作是…」「我对 X 指标负责」
- 数据驱动：每个手段以量化结果收尾
- 判断力叙事：讲「为什么这样判断」，不只讲「做了什么」
- 追问留白：结尾留钩子引导面试官追问

## 重要原则

- **绝不编造数据**：真实数据缺失时，AI 会用 `___` 占位并提醒你补充
- **经历库是单一可信源**：简历、逐字稿、定制版都从它生成，数据永远对齐
- **改了素材记得重新整理**：materials 更新后说一句「更新经历库」

## 小贴士
- 模板越详细，生成的风格越准——放你最满意的简历/话术当模板
- 经历库的数据越全，生成内容越可信
- 同一段经历可以生成简单版/复杂版多个逐字稿，面试时按时长灵活选用
